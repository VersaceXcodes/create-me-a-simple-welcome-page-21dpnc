**WHY**: This stack is standardized on Better Auth session cookies and email/password auth.

### Required Backend Pattern
```typescript
import crypto from 'crypto';
import { betterAuth } from 'better-auth';
import { toNodeHandler } from 'better-auth/node';


const betterAuthSecret =
  process.env.BETTER_AUTH_SECRET ||
  crypto.randomBytes(32).toString('base64');

const isTunnelEnvironment =
  Boolean(process.env.TUNNEL_DOMAIN) ||
  Boolean(process.env.EXPO_PUBLIC_API_URL?.includes('modal.host')) ||
  Boolean(process.env.EXPO_PUBLIC_API_URL?.includes('launchpulse.ai')) ||
  Boolean(process.env.VITE_API_BASE_URL?.includes('modal.host')) ||
  Boolean(process.env.VITE_API_BASE_URL?.includes('launchpulse.ai'));

export const auth = betterAuth({
  // pool or database adapter
  secret: betterAuthSecret,
  baseURL: process.env.BETTER_AUTH_URL || process.env.BACKEND_URL || 'http://localhost:3000',
  emailAndPassword: {
    enabled: true,
    autoSignIn: true,
    requireEmailVerification: false,
  },
  trustedOrigins: [
    `${process.env.EXPO_APP_SCHEME || 'launchpulse'}://*/*`,
    'exp://*/*',
    'exps://*/*',
    'http://localhost:*',
    'http://127.0.0.1:*',
    'https://*.launchpulse.ai',
    'https://*.modal.host',
    process.env.FRONTEND_URL,
    process.env.BACKEND_URL,
  ].filter(Boolean),
  advanced: {
    crossSubDomainCookies: {
      enabled: true,
    },
    trustedProxyHeaders: true,
    disableCSRFCheck: true,
    defaultCookieAttributes: (process.env.NODE_ENV === 'production' || isTunnelEnvironment)
      ? {
          sameSite: 'none',
          secure: true,
          partitioned: true,
        }
      : {
          sameSite: 'lax',
          secure: false,
        },
  },
});

app.all('/api/auth/*', toNodeHandler(auth));
```

These Better Auth cookie settings are required for LaunchPulse preview/tunnel auth. If login succeeds but the user stays signed out, this block is usually missing or wrong.

### Tunnel / Cookie Diagnosis (REQUIRED when auth is flaky)
- Identify the actual frontend origin and backend/auth origin you are testing before changing code
- Use localhost checks for local bundle/API debugging, but if auth requests target `https://*.modal.host` or `https://*.launchpulse.ai`, verify auth again through the actual frontend tunnel URL
- If localhost sign-in returns `200` but the session is still null while the auth base URL points at a tunnel, treat localhost as a potentially misleading auth environment rather than proof the app is broken
- Inspect Playwright network requests and `Set-Cookie` headers. Compare cookie domain, `sameSite`, `secure`, and request origin before changing auth architecture
- Do NOT switch to token auth just because localhost cannot persist cookies for a tunnel-hosted Better Auth backend

### Required Session Refresh Pattern
```typescript
import { useQueryClient } from '@tanstack/react-query';
import { SESSION_QUERY_KEY } from '@/lib/auth/use-session';

const queryClient = useQueryClient();

await authClient.signIn.email({ email, password });

const sessionResult = await authClient.getSession();
queryClient.setQueryData(SESSION_QUERY_KEY, sessionResult.data ?? null);

if (!sessionResult.data?.user) {
  throw new Error('Session was not established after sign-in');
}
```

### Required Session Query Pattern (avoid endless loading)
```typescript
const SESSION_TIMEOUT_MS = 10000;

const withTimeout = async <T>(promise: Promise<T>, timeoutMs: number): Promise<T> => {
  return await Promise.race([
    promise,
    new Promise<T>((_, reject) => {
      setTimeout(() => reject(new Error(`Session check timed out after ${timeoutMs}ms`)), timeoutMs);
    }),
  ]);
};

export const useSession = () =>
  useQuery({
    queryKey: SESSION_QUERY_KEY,
    queryFn: async () => {
      try {
        const result = await withTimeout(authClient.getSession(), SESSION_TIMEOUT_MS);
        return result.data ?? null;
      } catch (error) {
        console.warn('[auth] Session check failed, falling back to signed-out state:', error);
        return null;
      }
    },
    retry: false,
  });
```

Use a bounded timeout in `@/lib/auth/use-session` so iframe/cross-origin session failures fall back to signed-out state instead of leaving the app on "Checking your session..." forever.
